# 55bmw WordPress Content Site

This repository contains project notes for the 55bmw WordPress content site. The site is maintained as a lightweight content project using WordPress, Astra Pro, and Rank Math.

The purpose of this README is to give editors, developers, and maintainers a clear reference before making changes to the site structure, content, layout, or metadata.

## Project Overview

55bmw is handled as a WordPress content site. Most updates should be manageable from the WordPress dashboard, with theme-level layout settings handled through Astra Pro and page-level SEO fields handled through Rank Math.

The site should stay simple and easy to review. Any update should first make sense from a user and maintenance point of view before SEO fields or page wording are adjusted.

## Purpose of This Repository

This repository is used to document how the site should be reviewed and maintained. It is not meant to replace the WordPress dashboard or the live site.

Use this README when checking:

- page structure
- login-related wording
- Astra Pro layout behavior
- Rank Math metadata
- mobile spacing
- content update notes
- live reference alignment

The goal is to keep future edits consistent, especially when the site content or access-related pages are updated.

## Core Stack

The site uses WordPress as the content management system. Astra Pro is used for layout controls such as header behavior, footer structure, page width, typography, spacing, and responsive settings.

Rank Math is used for SEO-related fields such as page titles, meta descriptions, sitemap inclusion, canonical settings, and any schema options that are relevant to the page.

This setup is intentionally straightforward. The site does not need unnecessary plugin complexity unless a new tool solves a clear maintenance, publishing, security, or performance issue.

## WordPress Setup Notes

Content updates should be handled carefully from the WordPress dashboard. Before editing a page, check what the page is meant to do and whether the current structure still supports that purpose.

For login-related pages, the wording should stay clear and direct. The phrase 55bmw login may be used where the page is specifically discussing access or login reference handling, but it should not be repeated across unrelated sections.

Page content should be reviewed before metadata. If the visible page is unclear, changing the SEO title or description will not fix the underlying issue.

## Astra Pro Layout Notes

Astra Pro should handle shared layout settings whenever possible. Header spacing, mobile menu behavior, content width, typography, and footer layout should be managed through theme settings instead of one-off page styling.

Before publishing layout changes, preview the page on both desktop and mobile. Small spacing issues can make an access page feel unfinished, especially if buttons, menus, or form-related elements are affected.

Avoid adding local styling unless there is a specific reason. The more page-specific styling is added, the harder the site becomes to maintain later.

## Rank Math Notes

Rank Math should be used to keep metadata accurate and restrained. The SEO title and meta description should describe the page users actually see.

Do not force repeated keywords into the metadata. A plain, accurate title and description are usually better than a keyword-heavy version that feels unnatural.

After page edits, check the Rank Math fields again. The visible content, title tag, meta description, canonical URL, and sitemap settings should all point in the same direction.

## Content Review Notes

When updating content, start with the page itself. The heading, first paragraph, navigation labels, and any access-related wording should make sense before technical fields are adjusted.

A good update should answer three simple questions:

1. What is this page for?
2. What does the user need to understand here?
3. What should a maintainer check next time?

If a sentence does not help with one of those questions, it probably does not need to be there.

## Maintenance Notes

Review this README after major content edits, Astra Pro changes, Rank Math updates, plugin changes, or layout adjustments.

The site should stay easy to maintain. Avoid adding unnecessary sections, unsupported claims, or promotional wording. If a page starts to feel like a landing page instead of a clear content reference, rewrite it in a simpler maintenance-focused style.

## Update Checklist

Before finalizing a site update, check that:

- the visible page still explains its purpose clearly
- login-related wording is accurate and not overused
- Astra Pro layout settings still work on mobile
- Rank Math metadata matches the visible page
- the live reference still matches the notes in this repository
- no unsupported claims were added
- the content reads naturally

Keep changes small, clear, and easy for the next maintainer to understand.
